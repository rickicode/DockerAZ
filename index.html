1. Requirements

Before creating a Mimicry theme, ensure you understand:

  • Jinja2 Templating - Mimicry uses Jinja2 for all templates
  • HTML/CSS - Basic web development knowledge
  • JSON - For configuration files
üí° Recommended Bootstrap 5 or Tailwind CSS for styling. All themes should be responsive.

3. Required Files

3.1 schema.json

Contains theme metadata (name, version, author, etc.)

{
    "name": "My Theme",
    "version": "1.0.0",
    "author": "Your Name",
    "url": "https://yourwebsite.com"
}

3.2 widgets.json

Active widget configuration. Defines available widget zones and their widgets.

{
    "sidebar_right": [
        {
            "id": "recent_posts_1",
            "type": "recent_posts",
            "title": "Recent Posts",
            "settings": {"count": 5},
            "order": 0
        }
    ],
    "footer_main": []
}

3.3 widgets.defaults.json

Default widget configuration. Used when resetting widgets to defaults.

Structure is identical to widgets.json.

3.4 image.png

Theme preview image displayed in admin panel. Recommended size: 640x360px (16:9 aspect ratio).

4. Template Files

layout.html

Required

Base template that all other templates extend. Contains:

  • HTML head with meta tags & SEO
  • Navigation menu
  • Widget zones (header, footer)
  • Footer with legal links
{% raw %}<!DOCTYPE html>
<html lang="en">
<head>
    <title>{% block title %}{{ config.site_name }}{% endblock %}</title>
    <meta name="description" content="{% block meta_description %}{{ config.site_description }}{% endblock %}">
</head>
<body>
    {% from "themes/your-theme/widgets.html" import render_widget %}
    
    <!-- Navigation -->
    <nav>
        {% for item in menu_items %}
            <a href="{{ item.url }}">{{ item.title }}</a>
        {% endfor %}
    </nav>
    
    <!-- Main Content -->
    {% block content %}{% endblock %}
    
    <!-- Footer Widgets -->
    {% if widgets and widgets.footer_main %}
        {% for widget in widgets.footer_main %}
            {{ render_widget(widget) }}
        {% endfor %}
    {% endif %}
    
    <!-- Footer -->
    <footer>
        <a href="/page/terms-of-service">Terms of Service</a>
        <a href="/page/privacy-policy">Privacy Policy</a>
        <p>¬© {{ config.site_name }}</p>
    </footer>
</body>
</html>{% endraw %}

index.html

Required

Homepage template. Displays article listing.

{% raw %}{% extends "themes/your-theme/layout.html" %}

{% block content %}
    {% for article in articles %}
        <article>
            <a href="/{{ article.published_date.strftime('%Y/%m/%d') }}/{{ article.slug }}">
                <img src="{{ article.featured_image }}" alt="{{ article.title }}">
                <h2>{{ article.title }}</h2>
            </a>
            <p>{{ article.content|striptags|truncate(150) }}</p>
        </article>
    {% endfor %}
    
    <!-- Pagination -->
    {% include "themes/your-theme/partials/pagination.html" %}
{% endblock %}{% endraw %}

5. Widget System

5.1 widgets.html

Macro file for rendering widgets. Must implement render_widget macro.

{% raw %}{% macro render_widget(widget) %}
<div class="widget widget-{{ widget.type }}" id="widget-{{ widget.id }}">
    
    {% if widget.title %}
        <h5 class="widget-title">{{ widget.title }}</h5>
    {% endif %}

    {# Text Widget #}
    {% if widget.type == 'text' %}
        <div class="widget-content">{{ widget.content }}</div>

    {# HTML Widget #}
    {% elif widget.type == 'html' %}
        <div class="widget-content">{{ widget.content | safe }}</div>

    {# Recent Posts Widget #}
    {% elif widget.type == 'recent_posts' %}
        {% set posts = widget.render_data.posts if widget.render_data else [] %}
        {% for post in posts %}
            <a href="/{{ post.published_date.strftime('%Y/%m/%d') }}/{{ post.slug }}">
                {{ post.title }}
            </a>
        {% endfor %}

    {# Popular Posts Widget #}
    {% elif widget.type == 'popular_posts' %}
        {% set posts = widget.render_data.posts if widget.render_data else [] %}
        {% for post in posts %}
            <a href="/{{ post.published_date.strftime('%Y/%m/%d') }}/{{ post.slug }}">
                {{ post.title }}
            </a>
        {% endfor %}

    {# Categories Widget #}
    {% elif widget.type == 'categories' %}
        {% set cats = widget.render_data.categories if widget.render_data else [] %}
        {% for cat in cats %}
            <a href="/category/{{ cat.slug }}">{{ cat.name }}</a>
        {% endfor %}

    {# Search Widget #}
    {% elif widget.type == 'search' %}
        <form action="/" method="get">
            <input type="search" name="q" placeholder="Search...">
            <button type="submit">Search</button>
        </form>

    {# Tags Widget #}
    {% elif widget.type == 'tags' %}
        {% set tags = widget.render_data.tags if widget.render_data else [] %}
        {% for tag in tags %}
            <a href="/tag/{{ tag.name }}">#{{ tag.name }}</a>
        {% endfor %}

    {# Archives Widget #}
    {% elif widget.type == 'archives' %}
        {% set archives = widget.render_data.archives if widget.render_data else [] %}
        {% for arch in archives %}
            <a href="{{ arch.url }}">{{ arch.name }} ({{ arch.count }})</a>
        {% endfor %}

    {% endif %}
</div>
{% endmacro %}{% endraw %}

5.2 Widget Types

Type Description Data Available

html HTML content (use | safe) widget.content
recent_posts List of recent articles widget.render_data.posts
popular_posts Most viewed articles widget.render_data.posts
categories Category list with counts widget.render_data.categories
search Search form -
tags Tag cloud widget.render_data.tags
archives Monthly archives widget.render_data.archives

5.3 Widget Zones

Widget zones are defined in widgets.json. Common zones include:

  • sidebar_right - Right sidebar
  • sidebar_left - Left sidebar
  • header_top - Above header
  • content_bottom - Below main content (4-column layout)
  • footer_main - Inside footer (4-column layout)

7. Best Practices

‚úÖ Do
  • Use responsive design (mobile-first)
  • Include dark mode support if possible
  • Escape user content with | e filter
  • Use | safe only for trusted HTML content
  • Include legal page links in footer (/page/terms-of-service, /page/privacy-policy)
  • Implement all widget types in widgets.html
  • Use semantic HTML5 elements
  • Include Open Graph and Twitter Card meta tags
‚ùå Don't
  • Hardcode absolute URLs
  • Use inline styles excessively
  • Forget to handle empty states (no articles, no widgets)
  • Skip responsive testing
  • Ignore accessibility (a11y)

SEO Checklist

  • ‚úÖ Unique <title> per page
  • ‚úÖ Meta description per page
  • ‚úÖ Canonical URL: <link rel="canonical" href="...">
  • ‚úÖ Open Graph tags (og:title, og:image, etc.)
  • ‚úÖ Twitter Card tags
  • ‚úÖ Structured data (JSON-LD)
  • ‚úÖ Proper heading hierarchy (single <h1> per page)
